# BYOK Multi-Tenant Content Generation Platform (PoC)

A Proof of Concept (PoC) for a Multi-Tenant Content Generation Platform utilizing a "Bring Your Own Key" (BYOK) architecture. This application is designed to securely offload AI generation costs to users while providing a scalable, multi-tenant vector vault for PR and Marketing professionals.

## 🌟 Key Features

1. **BYOK & Key Verification:**
   - Users provide their own Gemini API Key directly in the UI.
   - The platform immediately validates the key using a lightweight `model.generate_content("ping")` test.
   - Ensures users pay API costs directly to Google, saving them from the high markups of traditional credit-based SaaS platforms.

2. **Multi-Tenant Vector Vault (PostgreSQL + pgvector):**
   - Secure tenant isolation using a `serialized_account_key` generated by hashing the user's API Key and Account ID (SHA-256).
   - Strict `WHERE` clauses ensure users can only ever retrieve their own Brand Voice documents and data.
   - Foundation for integrating RAG (Retrieval-Augmented Generation) capabilities natively using `pgvector`.

3. **Context Caching for Cost Optimization:**
   - Implements Google's Gemini Context Caching API natively. 
   - Ideal for analyzing large, long-form documents (e.g., source books, massive brochures) across multiple chats without repeatedly paying ingestion costs.

4. **Human-in-the-loop Editing:**
   - *AI Drafts, Human Refines*: Generated content is placed directly into an editable 'Artifacts' window for immediate professional review and finalization.

## 🛠️ Technology Stack

- **Frontend:** Streamlit
- **Backend/AI:** Google Gemini API (`google-genai` SDK)
- **Database:** PostgreSQL (with `pgvector` extension)

## 🚀 Quick Setup (Local Development)

### 1. Requirements
Ensure you have Python 3.10+ and a running PostgreSQL instance.

### 2. Database Initialization
Run the provided SQL script against your local PostgreSQL database to set up the necessary tables and extensions:
```bash
psql -U postgres -d your_db_name -a -f db_setup.sql
```

### 3. Environment Configuration
Create a `.env` file based on `.env.template` and add your database credentials:
```env
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your_password
POSTGRES_DB=vector_vault
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
```

### 4. Install Dependencies
```bash
pip install -r requirements.txt
```

### 5. Run the Application
```bash
streamlit run app.py
```

## 📂 Project Structure

- `app.py`: Main Streamlit application containing the UI, routing, logic, and API calls.
- `db_setup.sql`: Database initialization script for PostgreSQL routing and `pgvector`.
- `.env.template`: Template for environment variables.
- `requirements.txt`: Python package dependencies.
